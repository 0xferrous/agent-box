# Documentation Charter

This documentation set covers **Agent-box** and **Portal** as related but independent products.

## Audience

- Users who want to run coding agents in disposable containers (Agent-box)
- Users who want host capability mediation via a socket API (Portal standalone)
- Users combining both (Agent-box + Portal)
- Contributors and maintainers

## Goals

- Make first-time setup successful for each path
- Keep Portal clearly usable without Agent-box
- Separate learning, task execution, facts, and design rationale using Diátaxis
- Keep implementation details aligned with accepted ADRs

## Non-goals

- API stability guarantees beyond what is documented in reference pages
- Coverage of unrelated tooling outside Agent-box/Portal workflows

## Diátaxis policy

Each page has one primary mode:

- Tutorials: learning by doing
- How-to: task completion
- Reference: authoritative facts
- Explanation: concepts and trade-offs

Mixed-mode pages should be split.
